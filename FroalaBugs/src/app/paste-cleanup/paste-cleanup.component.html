Returning altered results in the paste.beforeCleanup event does not work

Clarification:
The first Froala editor fires the paste.beforeCleanup event. It calls a fuction which removes all images from the pasted html (see TypeScript code).
The second Froala editor fires the paste.afterCleanup event. It calls the same function as the first editor (see TypeScript code).
The result of the first editor still includes the image even though it's not present in the returned result (see console.log).
The result of the second editor has removed the image from the pasted result.

According to the docs (here and here) both events should pasted the 'new' returned string (From the docs: "If a string is returned, the new string will be used as the pasted content.")

Reproduction:

  1. Copy the text below the editors (including the image).
  2. Paste it into the first editor.
  3. The image is present in the paste result.
  4. Open the developer tools (F12).
  5. The returned result does not include the image tag. Why does the editor still show the image?
  1. Copy the text below the editors (including the image).
  2. Paste it into the first editor.
  3. The image is not present in the paste result.
  4. Open the developer tools (F12).
  5. The returned result does not include the image tag. Works as expected.
